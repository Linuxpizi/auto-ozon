Ozon 三个标识符的区别（官方文档定义）
我审查了你打开的 Ozon Seller API 中文文档页面，以下是 /v3/product/info/list 端点中对三个字段的官方定义**：

1. product_id（整数 int64）
Ozon系统中商品的标识符 — product_id**

这是 Ozon 平台内部为每个商品分配的唯一 ID
在卖家后台看不到，只能通过 API 获取
Ship API（POST /v4/posting/fbs/ship）的请求体要求传入 product_id
例如："product_id": 185479045

2. sku（整数 int64）
商品在 Ozon 系统中的标识符 — SKU

这是 Ozon 系统中的 SKU 编号
和 product_id 不同——一个 product 下可以有多个 SKU（不同变体/规格）
仅用于商品查询/创建，不用于订单操作

3. offer_id（字符串 string）
商品在卖家系统中的标识符 — 货号

这是卖家自己定义**的 SKU 编码（你的内部货号）
是字符串类型，不是整数
例如："offer_id": "MY-SKU-001"

从 /v3/product/info/list 请求体定义中：

|    字段    |           类型           |              官方描述               |
| :--------: | :----------------------: | :---------------------------------: |
|  offer_id  |     Array of strings     |   商品在卖家系统中的标识符 — 货号   |
| product_id | Array of strings <int64> | Ozon系统中商品的标识符 — product_id |
|    sku     | Array of strings <int64> |  商品在 Ozon 系统中的标识符 — SKU   |
