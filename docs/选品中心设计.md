# 选品中心重构 — 设计文档

## 1. 目标

重构选品中心前端,使 UI 更简洁、功能更聚焦、编辑体验更高效。

---

## 2. 范围变更

### 2.1 删除精铺管理 Tab
- 删除 `ProductSelectionView.vue` 中所有与 `activeTab === 'precision'` 相关的代码
- 包括:`precision-editor` 左右分栏编辑器、`precisionMode` / `precisionEdit` / `precisionOriginal` 等全部状态变量、`goToPrecision`、`fetchPrecisionProduct`、`savePrecision`、`syncTopSpecToFirstSku` 等函数、精铺管理相关的 CSS(`.precision-tab` / `.precision-editor` / `.precision-left` / `.precision-right` 等)
- **精铺功能后续将整合进新的编辑抽屉**(见 3.3 节),而非简单删除

### 2.2 保留采集商品列表
- 页面标题改为「选品中心」
- 仅保留一个 Tab 内容(可去掉 Tabs 组件,直接展示内容)

## 3. UI 重构方案

### 3.1 筛选栏重构

**现状问题**:筛选条件平铺在一行,所有条件都展开,视觉拥挤。

**新方案**:折叠式筛选栏 + 高级筛选

```
┌──────────────────────────────────────────────────────────┐
│ 🔍 [关键词搜索]          [平台▾] [评分▾]  [筛选▸] [重置] │
├──────────────────────────────────────────────────────────┤
│ 高级筛选(展开后):                                      │
│ [品牌] [分类] [价格区间: __ ~ __] [评论数: __ ~ __]     │
│ [折扣: __ ~ __]  [日期: __ ~ __]                        │
└──────────────────────────────────────────────────────────┘
```

- **默认行**:关键词搜索(输入框)、平台选择(下拉)、评分范围(下拉/滑块)、筛选按钮(展开/收起)、重置按钮、刷新按钮
- **展开行(高级筛选)**:品牌、分类、价格区间、评论数区间、折扣区间、日期范围
- 使用 `n-collapse-transition` 控制展开动画
- 筛选按钮旁显示当前激活的筛选数量 badge
- 删除选中按钮和批量删除移到表格工具栏

### 3.2 操作列精简

**现状**:操作列有多个按钮(详情、编辑、上传、删除、去精铺等)

**新方案**:仅保留 3 个操作

| 操作 | 图标 | 行为 |
|------|------|------|
| 编辑 | ✏️ | 打开编辑抽屉(左右联动面板) |
| 上传 | 🚀 | 打开上传弹窗 |
| 删除 | 🗑️ | Popconfirm 确认后删除 |

删除按钮使用 `n-button text type="error"` 纯图标样式,减少视觉噪音。

### 3.3 编辑抽屉 — 左右联动面板(核心重构)

**设计理念**:将「详情查看」和「编辑」合并为一个全屏抽屉,左侧展示原始采集数据(只读),右侧为优化/编辑后的数据。左右联动,配合 AI 翻译/优化按钮。

#### 3.3.2 联动机制

- **滚动联动**:左右面板同步滚动,当右侧滚动到某个模块时,左侧自动高亮对应区域(使用 `IntersectionObserver` 或滚动事件节流)
- **数据联动**:左侧只读展示原始值,右侧编辑值;右侧修改后如果与左侧不同,左侧对应字段显示「已修改」标记(绿色小点或下划线变色)
- **修改追踪**:维护 `originalSnapshot` 对比对象,通过 `isModified(field)` 判断字段是否被修改

#### 3.3.3 AI 翻译/优化按钮

每个文本模块右侧都有操作按钮:

| 模块 | 按钮 | 功能 |
|------|------|------|
| 商品名称 | [翻译] [AI优化] | 翻译:源语言→俄语;AI优化:润色标题提升搜索排名 |
| 商品描述 | [翻译] [AI优化] | 翻译:全文翻译;AI优化:优化描述文案 |
| 商品属性 | [翻译全部] [AI优化全部] + 每行 [翻译] [🗑️] | 整体操作或逐条翻译 |
| 图片 | [AI优化] [AI批量生成] | AI优化:优化单张图片;AI批量生成:自动生成场景图/白底图 |

按钮样式:`n-button size="small" secondary`,翻译按钮带 🌐 图标,AI优化按钮带 ✨ 图标。

#### 3.3.4 图片管理功能

右侧图片区域支持:

1. **自由删除**:每张图片右上角悬浮 ✕ 按钮,点击弹出确认后删除
2. **添加图片**:末尾显示 `+` 占位框,点击打开文件选择器或粘贴图片 URL
3. **AI 优化**:每张图片上悬浮 ✨ 按钮,调用 AI 接口优化图片(去背景/调整色调/提升清晰度)
4. **AI 批量生成**:图片区域顶部按钮,基于商品信息自动生成多张营销图
5. **拖拽排序**:支持图片拖拽调整顺序(可选,后续迭代)
6. **主图标记**:第一张图自动标记为「主图」

图片展示采用 Grid 布局:
```
┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐
│  主图    │ │  图2     │ │  图3     │ │  +添加   │
│  🌟 ✕   │ │  ✨ ✕   │ │  ✨ ✕   │ │         │
└─────────┘ └─────────┘ └─────────┘ └─────────┘
```

## 5. 数据流设计

### 5.1 编辑抽屉数据流

```
用户点击[编辑]
    │
    ▼
SelectionEditDrawer (接收 productId)
    │
    ├─ apiGet(`/selection/products/${id}`) → originalSnapshot (原始数据,只读)
    │
    └─ editableData = reactive({...originalSnapshot})  (可编辑副本)
           │
           ├─ EditPanelOriginal  ← originalSnapshot
           │
           └─ EditPanelOptimized ← editableData
                  │
                  ├─ AI翻译: apiPost(`/ai/translate`, {text, targetLang})
                  │    → 更新 editableData 对应字段
                  │
                  ├─ AI优化: apiPost(`/ai/optimize`, {text, type})
                  │    → 更新 editableData 对应字段
                  │
                  ├─ ImageManager ← editableData.images
                  │    ├─ 删除: editableData.images.splice(idx, 1)
                  │    ├─ 添加: editableData.images.push(newUrl)
                  │    ├─ AI优化: apiPost(`/ai/optimize-image`, {url})
                  │    └─ AI批量生成: apiPost(`/ai/generate-images`, {productInfo})
                  │
                  └─ [保存] → apiPut(`/selection/products/${id}`, editableData)
```
