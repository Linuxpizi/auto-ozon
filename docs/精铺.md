# [精铺](https://global-help.ozon.com/zh/products?region=CHN)

这里的精铺有两种方式
1. 通过复制 SKU 精铺，复制 Ozon
2. 其它平台，非 Ozon 店铺

## 主要处理的内容
1. 主图、其它详情图等进行处理(主要是本土化翻译)
2. 文案的翻译

### SKU 属性分类体系

所有采集到的 SKU 属性按以下 **8 大类** 分组展示：

#### 1. 📦 基础商品信息 (Basic Product Info)

| 字段名 | 来源 | 展示方式 | 可操作 |
|--------|------|---------|--------|
| `title` / `source_name` | 插件采集 / 手动输入 | 文本行 | 右侧可编辑 + AI翻译 |
| `brand` | 插件采集 | 文本行 | 只读（品牌需与Ozon库匹配） |
| `category` | 插件采集 | 面包屑路径 | 右侧可选择Ozon类目 |
| `seller_name` | 插件采集 | 文本行 | 只读 |
| `source_id` | 插件采集 | 文本行 | 只读 |
| `source_url` | 插件采集 | 可点击链接 | 只读 |

#### 2. 💰 价格与促销 (Price & Promotion)

| 字段名 | 来源 | 展示方式 | 可操作 |
|--------|------|---------|--------|
| `price` | 插件采集 | 金额 + 货币符号 | 右侧可输入定价 |
| `old_price` | 插件采集 | 划线价格 | 右侧可输入折扣前价格 |
| `discount` | 插件采集 | 百分比标签 | 只读（自动计算） |
| `currency` | 插件采集 | 货币代码 | 只读 |
| `vat` | — | — | 右侧可选择增值税率 |

#### 3. 📏 物理规格 (Physical Specifications)

| 字段名 | 来源 | 展示方式 | 可操作 |
|--------|------|---------|--------|
| `weight` / `spec_list[].weight_g` | 插件采集 | 数值 + 单位(g) | 右侧可输入 |
| `depth` / `spec_list[].depth_mm` | 插件采集 | 数值 + 单位(mm) | 右侧可输入 |
| `width` / `spec_list[].width_mm` | 插件采集 | 数值 + 单位(mm) | 右侧可输入 |
| `height` / `spec_list[].height_mm` | 插件采集 | 数值 + 单位(mm) | 右侧可输入 |

> **多变体场景**：若 `spec_list` 包含多个变体，展示为可展开的规格表格，每个变体行显示对应尺寸。

#### 4. ⭐ 评价与市场数据 (Reviews & Market Data)

| 字段名 | 来源 | 展示方式 | 可操作 |
|--------|------|---------|--------|
| `rating` | 插件采集 | ⭐ 评分 + 进度条 | 只读 |
| `review_count` | 插件采集 | 数字 + "条评论" | 只读 |
| `stock` | 插件采集 | 状态标签 | 只读 |

> **设计原则**：此分类为只读展示，用于帮助运营人员判断商品市场表现，辅助定价决策。

#### 5. 🖼️ 媒体资源 (Media Assets)

| 字段名 | 来源 | 展示方式 | 可操作 |
|--------|------|---------|--------|
| `images` / `source_images` | 插件采集 / 手动输入 | 缩略图网格 (4列) | 可拖拽排序、替换、删除 |
| `video_urls` | 插件采集 | 视频封面 + 播放图标 | 可预览 |

> **图片网格设计**：
> - 默认 4 列网格，每张图显示缩略图 + 序号角标
> - 主图（第一张）显示 "主图" 标签
> - Hover 显示操作按钮：放大、删除、设为主图
> - 图片数量角标显示在右上角

#### 6. 🏷️ SKU 变体 (SKU Variants)

| 字段名 | 来源 | 展示方式 | 可操作 |
|--------|------|---------|--------|
| `sku_list[].sku` | 插件采集 | 文本 | 只读 |
| `sku_list[].barcode` | 插件采集 | 文本 | 只读 |
| `spec_list[].color` | 插件采集 | 颜色标签 | 只读 |
| `spec_list[].size` | 插件采集 | 尺码标签 | 只读 |
| `offer_id` | Ozon API | 文本 | 只读 |

> **变体表格设计**：
> - 每个变体为一行，列包含：颜色、尺码、SKU、条形码、重量、尺寸
> - 若变体仅 1 个，简化为键值对展示

#### 7. 📝 通用属性 (Generic Attributes)

| 字段名 | 来源 | 展示方式 | 可操作 |
|--------|------|---------|--------|
| `attributes[].name` | 插件采集 | 键名 | 只读 |
| `attributes[].value` | 插件采集 | 键值 | 只读 |
| `source_attributes` | JSON存储 | 键值表格 | 右侧可编辑 + AI翻译 |
| `translated_attributes` | AI翻译 | 键值表格(目标语言) | 可手动修正 |

> **属性表格设计**：
> - 两列布局：`属性名 | 属性值`
> - 按属性重要性排序：材质 → 颜色 → 尺码 → 产地 → 其他
> - 右侧 AI 翻译后以双语对照形式展示（原文 / 译文）

#### 8. 🏪 Ozon 内部数据 (Ozon Internal Data)

| 字段名 | 来源 | 展示方式 | 可操作 |
|--------|------|---------|--------|
| `ozon_category_id` | Ozon API | 数字 | 只读（折叠展示） |
| `ozon_type_id` | Ozon API | 数字 | 只读（折叠展示） |
| `product_id` | Ozon API | 数字 | 只读（折叠展示） |
| `task_id` | Ozon API | 数字 | 只读（折叠展示） |

> **设计原则**：此分类默认折叠，点击 "Ozon内部数据" 展开查看。仅用于调试和排查问题。

## 实现方案

### 1. SKU 复制精铺

#### 流程
1. 输入源 SKU(Ozon 平台商品)
2. 调用 API 获取商品完整信息
3. 翻译处理:
   - 商品名称(俄语 → 目标语言)
   - 商品描述
   - 变体特征
4. 图片处理:
   - 下载原图
   - 可选:添加水印/翻译文字
5. 编辑优化后的信息
6. 创建新商品(草稿状态)
7. 提交审核

#### API 调用链
```
1. POST /v3/product/info/stocks    # 获取库存信息
2. POST /v4/product/info/list      # 获取商品列表详情
3. POST /v1/product/pictures/upload # 上传图片
4. POST /v2/product/import         # 创建商品
```

### 2. 链接爬取精铺

#### 流程
1. 输入商品链接(淘宝/1688/拼多多等)
2. 爬取商品信息:
   - 基本信息(标题、价格、规格)
   - 图片列表
   - SKU 变体
3. AI 翻译处理
4. 图片下载与处理
5. 手动编辑优化
6. 创建 Ozon 商品

### 2. 浏览器插件端类型定义 (`browser-extension/src/utils/types.ts`)

```typescript
interface ScrapedProduct {
  platform: 'ozon' | 'wb'
  sourceId: string           // 对应后端 source_id
  title: string
  price: number
  oldPrice: number           // 对应后端 old_price
  images: string[]
  rating: number
  reviewCount: number        // 对应后端 review_count
  brand: string
  category: string
  sellerName: string         // 对应后端 seller_name
  sellerUrl: string          // 对应后端 seller_url
  attributes: ProductAttribute[]
  description: string
  sourceUrl: string          // 对应后端 source_url
  scrapedAt: string          // ISO datetime, 对应后端 scraped_at

  videoUrls: string[]        // 视频 URL 列表
  skuList: Array<{ sku: string; barcode: string }>   // SKU+条形码
  specList: Array<{ weight_g: number; depth_mm: number; height_mm: number; width_mm: number; [key: string]: any }>

  ozonCategoryId: number     // 对应后端 ozon_category_id
  ozonTypeId: number         // 对应后端 ozon_type_id
}

interface ProductAttribute {
  name: string
  value: string
}
```

> **命名转换**：插件端使用 camelCase，后端统一用 snake_case。Schema 的 `model_validator` 自动做 `_camel_to_snake` 转换。

### 3. Ozon 商品详情页采集逻辑 (`ozon.content.ts`)

| 字段 | 采集方式 | DOM 选择器 / 策略 |
|------|---------|-------------------|
| `sourceId` | URL 路径末尾数字 | `/\/(\d+)\/?$/` |
| `title` | 标题元素 | `h1` / `[data-widget="webProductHeading"] span` |
| `price` | 当前价格 | `[data-widget="webPrice"] .tsHeadline600Large` → `parsePrice()` |
| `oldPrice` | 原价（划线价） | `[data-widget="webPrice"] span.pdp_bj` / `text-decoration: line-through` |
| `images` | 商品图库 | `[data-widget="webGallery"] img` → 替换 `/wc\d+/` 为 `/wc500/` 提升清晰度，最多 10 张 |
| `rating` | 评分 | `[data-widget="webReviewProductScore"]` 提取数字 |
| `reviewCount` | 评论数 | 同上，提取整数 |
| `brand` | 品牌 | `[data-widget="webBrandName"] a/span` / `a[href*="/brand/"]` |
| `category` | 面包屑 | `[data-widget="webBreadcrumb"] a` → `slice(1).join(' > ')` |
| `sellerName` | 卖家 | `[data-widget="webMerchantInfo"] a/span` / `[data-widget="webBestSeller"] button span` |
| `attributes` | 多策略属性提取 | 见下文 |
| `description` | 商品描述 | `[data-widget="webDescription"]` → 截取 2000 字符 |
| `sku` | Артикул 编号 | 正则匹配 `Артикул\|SKU` / `meta[itemprop="sku"]` / JSON-LD |
| `variants` | 变体选项 | `[data-widget="webOfferSelector"] button` → 逗号连接 |

**属性提取三策略（优先级递减）：**

1. **展开按钮 + 特征区域**：先点击 `показать все` 按钮展开折叠内容，然后查找 `[data-widget="webCharacteristics"]` 等区域内的名值对 div
2. **通用 div 结构**：遍历容器内 `div > div` 子元素，取前两个子节点作为 name/value
3. **dt/dd 结构**：查找 `dt` 元素及其 `nextElementSibling` 作为名值对

### 4. JSON-LD 补充数据

从 `<script type="application/ld+json">` 中提取 `@type === 'Product'` 的结构化数据：
- `sku` → 补充到 attributes
- `brand.name` → 备用品牌
- `weight` → 重量
- `color` → 颜色

### 5. 后端同步逻辑（Upsert）

```
POST /browser-sync/sync-products
```

- **新记录**：直接插入，`synced=True`
- **已存在**（按 `platform + source_id` 去重）：增量补全更新策略：
  - 标题/卖家/source_url：新值非空则覆盖
  - 品牌/品类/评分/评论数/属性/描述/图片：`_is_enriched()` 判断（新值非空/非零/非空列表）
  - 价格：新值非零则更新
  - discount/stock：新值非空则补全
  - JSON 数组字段（video_urls/sku_list/spec_list）：新值非空时合并更新
