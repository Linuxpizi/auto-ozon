# 商品管理

对归属的所有店铺商品进行同步

## 后端设计
1. 增加商品数据模型，需要展示归属的店铺
2. 增加商品同步的任务队列(分散时间、分散店铺、随机同步)
3. 在第二步后，同时同步 [商品信息](./api/根据标识符获取商品信息.md)，写入数据库

## UI 设计
1. 商品列表
   1. 产品 ID(product_id)
   3. Offer ID(offer_id)
   2. SKU Image
   4. SKU
   5. FBO 有库存(has_fbo_stocks)
   6. FBS 有库存(has_fbs_stocks)
   7. 是否归档(archived)
   8. 是否打折(is_discounted)
2. 操作列
   1. [归档商品](https://docs.ozon.ru/api/seller/zh/?__rr=3#operation/ProductAPI_ProductUpdateOfferID)
   2. [迁移](https://docs.ozon.ru/api/seller/zh/?__rr=3#operation/ProductAPI_ImportProductsBySKU)
   3. [备货](https://docs.ozon.ru/api/seller/zh/?__rr=3#operation/PostingAPI_ShipFbsPostingV4),tips 提示 **订单需在5天内完成备货，超时可能被Ozon自动取消**

## 开发记录

### 2026-06-19 实现

**需求理解**
- 商品列表展示所有店铺的商品（offer_id / product_id / SKU / 商品名 / 状态）
- 操作列支持「归档」和「迁移」两种操作
- 定时任务自动同步所有店铺的商品列表

**后端实现**

| 文件                                        | 改动说明                                                                                                                                                                  |
| ------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `backend/app/models/listing.py`             | Listing model 增加 `offer_id`、`product_id`、`archived_at` 字段                                                                                                           |
| `backend/app/schemas/listing.py`            | ListingBase/Create/Update/Read 同步增加 `offer_id`、`product_id`、`archived_at`                                                                                           |
| `backend/app/services/ozon_client.py`       | 新增 `get_product_list()` 游标分页获取全部商品；`archive_product()` 调用 `/v1/product/info/archives` 归档；`import_products_by_sku()` 调用 `/v1/product/info/import` 迁移 |
| `backend/app/services/sync_service.py`      | 新增 `sync_products_for_store()` 从 Ozon 同步商品并 upsert 到本地 listings 表                                                                                             |
| `backend/app/services/scheduler_service.py` | 注册默认定时任务 `sync_products`，每 12 小时执行                                                                                                                          |
| `backend/app/crud/listing.py`               | 新增 `archive_listing()`、`archive_listings_bulk()`、`get_listing_by_offer()`                                                                                             |
| `backend/app/api/routers/listing.py`        | 新增 `POST /listings/archive`（本地归档）、`POST /listings/archive/ozon`（Ozon 平台归档）、`POST /listings/import`（迁移）                                                |
| `backend/app/core/db.py`                    | SQLite 自动迁移补充 `offer_id`、`product_id`、`archived_at` 列                                                                                                            |

**前端实现**

| 文件                                           | 改动说明                                                             |

| `frontend/src/router/index.ts`                 | 新增 `/products` 路由                                                |

**API 接口**

| 方法 | 路径                         | 说明                     |
| ---- | ---------------------------- | ------------------------ |
| GET  | `/api/listings/`             | 商品列表（分页 + 筛选）  |
| GET  | `/api/listings/count`        | 商品总数                 |
| POST | `/api/listings/archive`      | 本地批量归档             |
| POST | `/api/listings/archive/ozon` | Ozon 平台归档 + 本地归档 |
| POST | `/api/listings/import`       | Ozon 商品迁移            |
