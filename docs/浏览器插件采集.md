# 浏览器插件采集

## 一、概述

浏览器插件是 auto-ozon 项目的数据采集入口之一。与后端 scraper 受限于反爬机制不同，浏览器插件运行在用户真实的浏览器环境中，能够直接访问 Ozon / WB 页面的 DOM，绕过服务端反爬限制，采集更完整的商品数据。

### 核心目标

1. **采集 Ozon / WB 商品数据** — 在用户浏览商品页面时，自动或手动提取商品信息
2. **国内平台比价** — 将采集到的商品数据发送到后端，在 1688/淘宝/拼多多等平台匹配相似款
3. **筛选过滤** — 用户可设定采集条件（价格区间、销量、评分等），只采集符合要求的商品
4. **批量导出/上架** — 将筛选后的商品数据同步到后端，用于精铺上架

## 二、技术方案

### 2.1 技术选型

| 项目     | 方案                                     |
|----------|------------------------------------------|
| 插件标准 | Chrome Manifest V3                        |
| UI 框架  | 纯 HTML/CSS/JS（轻量，无框架依赖）        |
| 数据存储 | Chrome Storage API（本地）+ 后端同步       |
| 通信方式 | content script ↔ background ↔ popup       |
| 后端通信 | background service worker 通过 fetch 调用 backend REST API |

### 2.3 数据流

```
用户浏览 Ozon/WB 商品页
    │
    ▼
Content Script 提取页面数据 (DOM 解析)
    │ chrome.runtime.sendMessage
    ▼
Background Worker 接收数据
    │
    ├──► 本地 Chrome Storage 暂存
    │
    └──► fetch → Backend /api/browser-sync/sync-products
              │
              ▼
         Backend 存入数据库
              │
              └──► 后续比价/精铺流程
```

## 三、功能模块

### 3.1 Content Script — Ozon 页面采集

采集字段：

| 字段           | 说明                 | 采集方式                 |
|----------------|----------------------|--------------------------|
| source_id      | 商品 ID              | URL 路径提取             |
| title          | 商品标题             | DOM: `h1` 或 `span[data-widget="webProductHeading"]` |
| price          | 当前价格             | DOM: 价格元素            |
| old_price      | 原价（折扣前）       | DOM: 删除线价格          |
| images         | 商品图片列表         | `img` 标签提取           |
| rating         | 评分                 | DOM 评分元素             |
| review_count   | 评价数量             | DOM 评价数量             |
| brand          | 品牌                 | 属性列表提取             |
| category       | 面包屑导航 → 类目    | 面包屑元素               |
| seller_name    | 卖家名称             | 卖家信息区域             |
| attributes     | 商品属性列表         | 规格属性区域             |
| description    | 商品描述             | 描述区域                 |
| url            | 商品链接             | `window.location.href`  |
| scraped_at     | 采集时间             | 自动填充                 |

### 3.2 Content Script — WB (Wildberries) 页面采集

采集字段：

| 字段           | 说明                 | 采集方式                 |
|----------------|----------------------|--------------------------|
| source_id      | 商品 ID (nmId)       | URL 路径提取             |
| title          | 商品标题             | DOM: 商品名称元素         |
| price          | 当前价格             | DOM: 价格元素            |
| old_price      | 原价                 | DOM: 旧价格元素          |
| images         | 商品图片列表         | `img` 标签               |
| rating         | 评分                 | DOM 评分元素             |
| review_count   | 评价数量             | DOM 评价数量             |
| brand          | 品牌                 | 属性列表                 |
| category       | 类目                 | 面包屑导航               |
| seller_name    | 卖家名称（供应商名） | 卖家信息区域             |
| attributes     | 商品属性             | 规格区域                 |
| description    | 商品描述             | 描述区域                 |
| url            | 商品链接             | `window.location.href`  |

### 3.3 Popup UI

主要面板：

1. **采集面板**
   - 显示当前页面是否可采集（Ozon/WB 识别）
   - 「一键采集」按钮
   - 实时预览采集到的数据

2. **采集记录**
   - 已采集商品列表（来自 Chrome Storage）
   - 支持删除、编辑
   - 「同步到后端」按钮

3. **设置面板**
   - Backend API 地址配置（默认 `http://localhost:9000`）
   - 自动采集开关（进入 Ozon/WB 商品页时自动采集）
   - 采集过滤条件：
     - 价格区间（RUB）
     - 最低评分
     - 最低评价数量
     - 排除品牌（黑名单）
     - 包含品牌（白名单）
   - 批量操作设置

### 3.4 Background Service Worker

职责：

1. **消息中转** — 接收 content script 数据，转发给 popup
2. **后端同步** — 将采集数据 POST 到后端 API
3. **自动采集触发** — 监听 tab 更新，匹配 Ozon/WB URL 时通知 content script
4. **Badge 更新** — 显示未同步商品数量

## 四、Backend API 扩展

### 4.1 新增路由

在后端新增 `browser_sync` 路由模块，接收浏览器插件推送的商品数据。

| 方法   | 路径                                    | 说明                     |
|--------|-----------------------------------------|--------------------------|
| `POST` | `/api/browser-sync/sync-products`       | 批量同步采集的商品数据    |
| `GET`  | `/api/browser-sync/products`            | 获取已同步的采集商品列表  |
| `DELETE`| `/api/browser-sync/products/{id}`      | 删除采集商品记录          |
| `POST` | `/api/browser-sync/match-suppliers`    | 触发比价匹配（查国内供应链）|

### 4.2 数据模型

```python
class ScrapedProductRecord(Base):
    """浏览器插件采集的商品记录"""
    __tablename__ = "scraped_product_records"

    id = Column(Integer, primary_key=True)
    platform = Column(String)        # "ozon" | "wb"
    source_id = Column(String)       # 平台商品 ID
    title = Column(String)
    price = Column(Float)            # 当前价格（RUB）
    old_price = Column(Float)        # 原价
    images = Column(JSON)            # 图片 URL 列表
    rating = Column(Float)
    review_count = Column(Integer)
    brand = Column(String)
    category = Column(String)
    seller_name = Column(String)
    seller_url = Column(String)
    attributes = Column(JSON)        # 属性列表
    description = Column(Text)
    source_url = Column(String)      # 原始商品链接
    scraped_at = Column(DateTime)    # 采集时间
    synced = Column(Boolean, default=False)  # 是否已同步到后端
    matched = Column(Boolean, default=False)  # 是否已比价匹配
    matched_suppliers = Column(JSON)  # 匹配到的供应商信息
    created_at = Column(DateTime, server_default=func.now())
    updated_at = Column(DateTime, onupdate=func.now())
```
