# 订单管理功能文档

对授权的所有店铺订单信息进行同步管理

## 订单列表

1. 检索条件
   1. 店铺名称
   2. 货件状态
   3. 货件号/订单号/运单号
   4. 刷新
   5. 一键同步
   6. 总数(只展示)
2. 订单列表(支持翻页)
   1. 图片/货件号/SKU
      1. 包装在一起
         1. 左侧图片(商品管理已有这些信息)
         2. 右侧上货件号
         3. 右侧下 SKU
      2. 点击图片，可以放大图片
   2. 订单号
   3. 状态/操作
   4. 必须发货/商品名称
      1. 包装起来
         1. 上面部分显示必须发货(最晚发货日期)
         2. 下面部分显示商品名称
   5. 运单号/数量/成交价
   6. 店铺
   7. 操作列
      1. [备货](https://docs.ozon.ru/api/seller/zh/?__rr=3#operation/PostingAPI_ShipFbsPostingV4),tips 提示 **订单需在5天内完成备货，超时可能被Ozon自动取消**, 状态是待包装的可以点击备货，其余状态置灰
## 订单

### A.2 备货接口 `POST /v4/posting/fbs/ship`

[官方文档](https://docs.ozon.ru/api/seller/zh/#operation/PostingAPI_ShipFbsPostingV4)

> **重要（来自本地 docs）：** 状态码为 200 的响应不代表订单已成功备货。请使用 `/v3/posting/fbs/get` 方式来检查，订单是否已完成备货。如果响应中包含 `result.substatus = ship_failed`，请重新为订单备货。

**请求**

```json
{
  "posting_number": "89491381-0072-1",
  "packages": [
    {
      "products": [
        {
          "product_id": 185479045,
          "quantity": 1
        }
      ]
    }
  ],
  "with": {
    "additional_data": true
  }
}
```

**响应**

```json
{
  "additional_data": [
    {
      "posting_number": "89491381-0072-1",
      "products": [
        {
          "currency_code": "RUB",
          "mandatory_mark": ["123"],
          "name": "string",
          "offer_id": "17125",
          "price": "2000",
          "quantity": 1,
          "sku": 149618972
        }
      ]
    }
  ],
  "result": ["89491381-0072-1"]
}
```

## D. 同步流程

定义在 `backend/app/services/sync_service.py` → `sync_orders_for_store()`。

1. 创建 `OzonClient`，获取 30 天内 + 未来 5 天的截止时间窗口
2. 拉取全部 10 种状态（`awaiting_registration` 到 `not_accepted`）
3. 游标分页（每页 100 条），遍历所有数据
4. 从 `listings` 表批量查询 `image_url`（按 `offer_id` 匹配）
5. 将 `shipment_date` / `in_process_at` 解析为 datetime 写入对应字段
6. GMV 优先使用 `customer_price`，回退到 `price * quantity`
7. 提取 `financial_data`：`payout`、`customer_price`、`commission`、`discount`
8. 提取配送信息：`delivery_method.name`、`delivery_method.warehouse`
9. 提取 `barcodes`：`upper_barcode`、`lower_barcode`
