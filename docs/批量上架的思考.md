# [Ozon 批量上架的思考](https://docs.ozon.ru/api/seller/zh/?__rr=1#tag/ProductAPI)

1. 必须要严格按个阅读[商品上传和更新](https://docs.ozon.ru/api/seller/zh/?__rr=1#tag/ProductAPI),可以用浏览器打开。这一条必须要严格执行，并且要非常细致的阅读理解。
2. 然后思考下面的相关问题，给出可行开发计划，以及可操作的 UI
3. 切记重要的一点，思考是否可以批量执行，当然每一条手工操作，精铺也是需要的。但是，思考自动化批量操作放在第一位。

## Ozon 选品

1. 标题从新优化
2. 主图优化，以图生图，因为都已经是 Ozon 上架的产品
3. 做上面操作的目的就是防止同质化
4. 这里就必须要增加一个功能 1688 找同款，确保用户在 Ozon 下单后，我们在 1688 一定可以采购

所以，这里的流程就是
1. Ozon 采集数据
2. 1688 找同款
3. 按照上述操作优化
4. 然后产品参数比如规格，重量等，需要参考 1688 的重新设定
5. 定价
6. 上架到 Ozon 店铺

## 1688 选品

1. 标题翻译
2. 图片
   1. 在 Ozon 相似的，使用 1688 的主图替换 Ozon 主图的主体
   2. 完全使用 1688 的，需要对图片重新生成 + 内容本土化翻译(如果主图有文字)

所以，这里的流程就是
1. 1688 采集数据
2. 按照上述操作优化
3. 产品规格，重量等必要的参数必须要有且正确
4. 定价
5. 上架
