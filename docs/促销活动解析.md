### 一、 核心总览：促销活动的两大阵营

Ozon的促销体系分为**官方发起的活动**和**卖家自建的促销**。无论哪一种，折扣成本均由卖家承担，但**流量逻辑截然不同**。

| 类别         | **平台官方活动**                               | **卖家自主促销**                                 |
| :----------- | :--------------------------------------------- | :----------------------------------------------- |
| **发起方**   | Ozon平台（如双十一、黑五、季节性大促）         | 卖家自己在后台创建                               |
| **参与门槛** | 需报名，平台审核（对商品评分、价格历史有要求） | 无需审核，随时开启                               |
| **流量加成** | **极高**（首页曝光、专属标签、站外导流）       | **一般**（仅靠店铺内部引流，不如官方活动权重高） |
| **典型形式** | 库存折扣、超级品类日                           | 多级满减、普通折扣券、促销码                     |

**关键铁律（必须牢记）**：如果一个商品同时命中了多个促销规则，**系统不会叠加优惠，而是自动取所有可用价格中的最低价**进行展示和销售。

---

### 二、 深度拆解：多级满减（多级满额折扣）

这是卖家自建促销中最核心的**“客单价拉升利器”**。它的本质是**订单级折扣**，而不是单品级折扣。

#### 1. 运行机制
-   **触发条件**：基于买家购物车内**最终支付的商品总金额**（不含运费）。
-   **阶梯设置**：支持设置 **2~4个阶梯**。例如：满2000卢布减200；满5000卢布减700；满10000减2000。
-   **适用范围**：一旦开启，**自动覆盖该卖家店铺内所有符合参加条件的商品**（除非单独排除个别商品）。买家只需在店内凑单，系统会自动扣减。

#### 2. 折扣形式
-   **固定金额减免**（如减300卢布）：适合中高客单价商品，买家感知清晰。
-   **百分比折扣**（如打9折）：适合低价商品，能灵活控制成本。

#### 3. 实战策略与陷阱
-   **核心用途**：诱使买家购买本不需要的“凑单品”以达到满减门槛。建议搭配低价配件（如手机壳配充电器）一起销售。
-   **成本测算**：务必计算**边际利润**。如果利润率只有20%，设置满10000减2500（即75折）就会亏损，必须用财务模型核算。
-   **最大陷阱**：此折扣会与**平台大促（如11.11）的库存折扣联动**。假如你的商品在官方大促中打了7折，同时你开着“满10000减3000”（相当于7折），两者不叠加，系统会取其中一个**更低**的价格，通常是官方大促价或更低的那个。

### 三、 深度拆解：库存折扣（特价清仓）

这是**平台官方活动**最常用的载体，也是大促期间的绝对主力。

#### 1. 运行机制
-   **核心定义**：卖家为特定数量的库存设定一个**限时特价**。
-   **数量锁定**：你必须明确指定参与促销的库存件数（比如只拿100件出来打5折）。
-   **自动恢复**：当这100件卖完，或者促销时间截止，商品**自动、立即**恢复原价，无需手动干预。

#### 2. 价格展示逻辑
-   促销期间，前台展示的是**红色划线的促销价**（原价被划掉），视觉冲击力强。
-   平台算法会对参与库存折扣的商品进行**流量加权**，因为平台认为这是“高转化潜力”商品。

#### 3. 实战策略与陷阱
-   **核心用途**：**测款**（用少量库存折扣看市场反应）、**清仓**（季节末尾清掉过季品）、**冲排名**（短期拉高类目排名）。
-   **申报门槛**：Ozon官方对折扣力度有最低要求（通常要求比近30天均价低10%~15%以上）。折扣力度不够，报名会被驳回。
-   **关于“不叠加”的深度解释**：如果你的A商品同时报名了“双11大促（8折）”和“黑五清仓（7折）”，因为不叠加，最终只会展示**7折**的价格。注意，促销库存数量**不会合并**，哪个先卖完就结束哪个活动。

### 四、 终极避坑指南：三大促销规则的叠加与排他性

这是新手卖家最容易亏损的地方，务必逐条核对：

| 规则场景                              | 是否叠加？   | 最终执行逻辑                                                                                     |
| :------------------------------------ | :----------- | :----------------------------------------------------------------------------------------------- |
| **多个库存折扣活动**（如A大促+B大促） | ❌ **不叠加** | 取**折扣最大**（即价格最低）的那个活动价生效。                                                   |
| **库存折扣 + 卖家多级满减**           | ❌ **不叠加** | 系统计算时，会先看单品折后价，再看订单满减总额，**只取最终支付总额更低的那个方案**，不会折上折。 |
| **库存折扣 + 普通商品优惠券**         | ❌ **不叠加** | **促销码（优惠券）按促销活动中的最低价生效**。如果库存折扣价已经很低，优惠券可能无法使用。       |
| **多级满减 + 店铺优惠券**             | ❌ **不叠加** | 同样取最低，不会让买家同时享受“满减”又用“优惠券”抵扣（除非平台另有特殊说明）。                   |

### 五、 卖家实操决策树（什么时候用什么工具？）

1.  **想清掉换季尾货，不求利润只求回款**：**首选库存折扣**。直接给底价，限量供应，利用平台流量迅速清空。
2.  **想打爆新品，提升店铺权重**：**先用库存折扣**（低价冲量拿类目排名），等排名稳定后，**转用多级满减**（把流量转化为高客单价订单）。
3.  **想提升客单价，让买一件的变成买三件**：**只用多级满减**。关闭其他重叠折扣，避免价格体系混乱。
4.  **想配合节日拉新**：设置**中等力度的库存折扣**（如85折），配合官方活动标签，获取免费曝光流量。

### 六、 财务风控终极提醒（算总账）

Ozon结算时，**促销折扣费用全部从卖家收入中扣除**，平台仅收取佣金（按原价或折后价，视具体合同而定，通常按实际成交价抽成）。

**错误示例**：一个商品成本1000卢布，原价1500卢布。你报名了**官方库存折扣7折（1050卢布）**，同时你忘了关掉**多级满减（满1000减200）**。
结果：系统取最低价，最终成交价为 **1050 - 200 = 850卢布**？**错！** 因为是取**两个方案最终支付总额**的最低值。方案一（库存折扣）=1050卢布；方案二（满减）=1500-200=1300卢布。系统选**1050卢布**成交。减去1000成本，你只赚50卢布，如果算上物流和佣金，**必然亏损**。

**结论**：务必在每次大促前，使用后台的“**促销日历**”和“**价格计算器**”，核对所有生效的活动，确保最低成交价仍然覆盖你的“盈亏平衡线”。
