# 卖家评级

## [参考文档](./api/获取错误指数：FBS%20和%20rFBS.md)

## 后端、数据模型

1. 店铺信息增加卖家评级相关的数据(JSON)
2. 卖家的评级信息，可以每天定时任务进行同步

## UI 设计
在店铺管理列表，新增卖家评级列
    点击后，小开片展示卖家的评级信息

---

## 开发记录

### 2026-06-19 实现

**后端**
- `Store` model 增加 `seller_rating` Text 字段，存储 JSON 字符串
- `schemas/store.py` 的 `StoreBase`、`StoreUpdate`、`StoreRead` 同步增加 `seller_rating` 字段
- `OzonClient` 增加 `get_seller_rating()` 方法，调用 `POST /v1/rating/index/fbs/info`
- `sync_service.py` 增加 `sync_seller_rating_for_store()`，将评分写入 Store.seller_rating
- `scheduler_service.py` 注册默认定时任务 `sync_seller_rating`，每天执行一次
- `core/db.py` 增加 SQLite 自动迁移，确保已有表补充新列

**前端**
- `store/index.ts` 的 `StoreItem` 接口增加 `seller_rating` 字段
- `StoreList.vue` 在表格中新增「卖家评级」列
- 点击评级数字弹出 Popover，展示：评级指数、币种、周期、处理费用、缺陷记录
