## 二、电商平台输出尺寸规范

### 2.1 Ozon 平台图片尺寸要求

| 图片类型 | 推荐尺寸 | 比例 | 最大文件大小 | 格式 |
|----------|----------|------|-------------|------|
| 商品主图 | 900×1200 px | 3:4 | 5 MB | JPG/PNG |
| 商品辅图 | 900×1200 px | 3:4 | 5 MB | JPG/PNG |
| 详情图(横) | 1200×900 px | 4:3 | 5 MB | JPG/PNG |
| 详情图(方) | 1000×1000 px | 1:1 | 5 MB | JPG/PNG |
| Banner/视频封面 | 1200×675 px | 16:9 | 5 MB | JPG/PNG |

### 2.2 GPT Image API 尺寸映射

GPT Image API 支持以下尺寸，我们需要映射到电商规范：

| GPT API 尺寸 | 实际输出 | 适配场景 | 处理方式 |
|-------------|----------|----------|----------|
| 1024×1024 | 1:1 正方形 | 详情图(方) | 裁剪/缩放到 1000×1000 |
| 1024×1536 | 2:3 竖版 | 主图/辅图 | 缩放到 900×1350 → 裁剪 900×1200 |
| 1536×1024 | 3:2 横版 | 详情图(横) | 缩放到 1200×800 → 裁剪到 1200×900 |
| 1536×1536 | 1:1 高清 | 详情图(方) | 缩放到 1000×1000 |

### 2.4 自动裁剪/缩放流程

```
GPT Image API 输出 (1024×1536)
    │
    ▼
后端 image_service.py 自动处理
    ├─ 1. 缩放到目标宽度 (保持比例)
    ├─ 2. 居中裁剪到目标高度
    ├─ 3. 压缩到 < 5MB (quality=90 for JPG)
    └─ 4. 保存到 static/images/
    │
    ▼
返回处理后的 URL + 原始 URL
```

### 3.2 编辑工具交互流程

#### 工具一：自然语言编辑(核心)

```
用户输入: "把背景换成纯白色"
    │
    ▼
前端: 可选 — 用户框选编辑区域(虚线矩形)
    │  如果不框选 → 全图编辑
    │  如果框选 → 生成 mask → 传入 API
    ▼
后端: POST /api/v1/image/edit
    {
      image_url: "...",
      prompt: "把背景换成纯白色",
      mask: "base64..."  (可选),
      output_preset: "ozon_main"
    }
    │
    ▼
OpenAI API: client.images.edit(model, image, prompt, mask, size)
    │
    ▼
后端: 自动裁剪 → 返回 result_url + version_id
    │
    ▼
前端: 更新画布 + 推入版本栈
```

#### 工具二：框选删除物体

```
用户操作:
    1. 点击 [删物体] 按钮
    2. 在图片上拖拽画出矩形框(或自由画笔涂抹)
    3. 生成半透明红色遮罩
    4. 点击 [执行删除]
    │
    ▼
前端: 将遮罩区域转为 mask image (黑色=保留, 白色=编辑区域)
    │
    ▼
后端: POST /api/v1/image/edit
    {
      image_url: "...",
      prompt: "Remove the selected object seamlessly, fill with surrounding background",
      mask: "base64...",
      output_preset: "ozon_main"
    }
```

#### 工具三：去背景

```
用户操作: 点击 [去背景] → 选择底色(白/透明)
    │
    ▼
后端: POST /api/v1/image/edit
    {
      image_url: "...",
      prompt: "Remove the background completely. Output product on [white/transparent] background. Keep the product intact."
    }
```

#### 工具四：AI 扩图

```
用户操作: 点击 [AI扩图] → 选择扩展方向(四周/左右/上下)
    │
    ▼
前端: 计算扩展后的目标尺寸
    │
    ▼
后端: POST /api/v1/image/expand
    {
      image_url: "...",
      direction: "all",       // all/left/right/top/bottom
      expand_ratio: 0.5,      // 扩展比例
      prompt: "Extended product photo, matching original style",
      output_preset: "ozon_detail_h"
    }
```

#### 工具五：高清修复

```
用户操作: 点击 [高清修复] → 选择倍率(2x/4x)
    │
    ▼
后端: POST /api/v1/image/upscale
    {
      image_url: "...",
      scale: 2,              // 2 或 4
      output_preset: "ozon_main"
    }
    │
    ▼
方案A: GPT Image Edit — "Upscale this image to higher resolution, enhance details and sharpness"
方案B: Real-ESRGAN (PowerPaint 环境已有) — 本地 4x 超分
```

## 四、后端 API 设计

### 4.1 统一编辑端点

```python
# POST /api/v1/image/edit — 核心编辑接口
class ImageEditRequest(BaseModel):
    image_url: str                           # 原始图片 URL
    prompt: str                              # 编辑指令(自然语言)
    mask: Optional[str] = None               # 遮罩 base64 (白色=编辑区域)
    output_preset: str = "ozon_main"         # 输出尺寸预设
    custom_width: Optional[int] = None       # 自定义宽度
    custom_height: Optional[int] = None      # 自定义高度
    quality: int = 90                        # 输出质量 1-100

class ImageEditResponse(BaseModel):
    original_url: str
    result_url: str                          # 处理后图片 URL
    version_id: str                          # 版本 ID
    output_size: str                         # "900x1200"
    file_size_kb: int                        # 文件大小
```

### 4.2 扩图端点

```python
# POST /api/v1/image/expand — AI 扩图
class ImageExpandRequest(BaseModel):
    image_url: str
    direction: str = "all"                   # all/left/right/top/bottom
    expand_ratio: float = 0.5                # 扩展比例 (0.25-1.0)
    prompt: Optional[str] = None             # 可选: 引导生成方向
    output_preset: str = "ozon_detail_h"

class ImageExpandResponse(BaseModel):
    original_url: str
    result_url: str
    version_id: str
    original_size: str                       # "900x600"
    expanded_size: str                       # "1200x900"
```

## 六、版本管理状态设计

```typescript
// ImageEditor.vue 内部状态
interface VersionNode {
  version_id: string;
  description: string;
  file: string;
  url: string;           // /static/image_versions/xxx/v2.png
  prompt: string | null;
  timestamp: string;
  parent_version: string | null;
  output_size: string;
}

const state = reactive({
  // 当前图片
  currentImageUrl: '',

  // 版本管理
  versions: [] as VersionNode[],
  currentVersionIndex: -1,

  // 编辑状态
  tool: 'prompt' as 'prompt' | 'brush' | 'rect' | 'remove-bg' | 'upscale' | 'expand',
  editPrompt: '',
  maskData: null as string | null,   // base64 mask
  brushSize: 20,

  // 输出设置
  outputPreset: 'ozon_main',
  outputQuality: 90,

  // UI 状态
  processing: false,
  zoom: 1,
  panX: 0,
  panY: 0,
  showCompare: false,
});

// 关键方法
function pushVersion(node: VersionNode) {
  // 截断后续版本(如果从中间版本编辑)
  state.versions = state.versions.slice(0, state.currentVersionIndex + 1);
  state.versions.push(node);
  state.currentVersionIndex = state.versions.length - 1;
}

function undo() {
  if (state.currentVersionIndex > 0) {
    state.currentVersionIndex--;
    state.currentImageUrl = state.versions[state.currentVersionIndex].url;
  }
}

function redo() {
  if (state.currentVersionIndex < state.versions.length - 1) {
    state.currentVersionIndex++;
    state.currentImageUrl = state.versions[state.currentVersionIndex].url;
  }
}

function restoreVersion(index: number) {
  state.currentVersionIndex = index;
  state.currentImageUrl = state.versions[index].url;
}
```
